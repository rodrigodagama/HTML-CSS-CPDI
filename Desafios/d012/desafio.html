Desafio + Revisão de conteúdo

Desafio + Revisão de conteúdo:

Implementação de um jogo inspirado em "Top Gear" em 2D, onde o jogador controla um carro (McQueen) e deve evitar colidir com carros inimigos (Hiks) enquanto coleta pontos.

Configuração da Física:

A biblioteca de física é chamada com require ("physics"), e o motor de física é iniciado com physics.start().

A gravidade é definida como (0, 0) com physics.setGravity(0, 0).

O modo de renderização é definido como "normal" com physics.setDrawMode("normal"). Outros modos incluem "normal" e "debug" para depuração.

Remoção da Barra de Notificações:

A barra de status do dispositivo é ocultada para criar uma experiência de tela cheia.

Criação de Grupos de Exibição:

Três grupos são criados: grupoBg para elementos de fundo, grupoMain para elementos principais do jogo e grupoUI para a interface do usuário.

Variáveis de Pontuação e Vidas:

Duas variáveis, pontos e vidas, são inicializadas para rastrear a pontuação do jogador e suas vidas restantes.

Background:

Um plano de fundo é criado com uma imagem ("bg2.png") no grupo grupoBg, e sua posição é centralizada.

Música de Fundo:

Uma música de fundo ("topgear.mp3") é carregada e reproduzida em loop infinito com um volume ajustado.

Paredes Invisíveis:

Uma parede invisível é criada à esquerda da tela para evitar que o carro do jogador saia da área de jogo.

Placar:

Dois objetos de texto são criados no grupoUI para exibir os pontos e as vidas do jogador.

Carro do Jogador (McQueen):

Uma imagem do carro do jogador é criada no grupoMain

O carro é configurado como um corpo físico "kinematic" para controle direto do jogador.

Botões de Controle:

Dois botões (cima e baixo) são criados para permitir que o jogador mova o carro para cima e para baixo.

Funções de movimentação (cima() e baixo()) são definidas para responder ao toque nesses botões.

Carro Inimigo (Hiks):

Uma função inimigoAtirar() é definida para criar tiros do inimigo.

A função cria um tiro, configura sua trajetória e remove-o após um período de tempo.

Movimentação do Inimigo:

Um carro inimigo é criado e seu movimento é controlado por transições.

O carro inimigo é configurado como um corpo físico "dynamic" com detecção de colisão como sensor.

Função de Colisão:

A função onCollision() trata colisões entre o carro do jogador, o carro inimigo e as paredes.

Quando há colisões, as vidas do jogador diminuem e os pontos aumentam.

Listeners de Colisão:

Os ouvintes de colisão são ativados para detectar colisões durante o jogo.

Criação e Movimento do Carro Inimigo:

As funções criarCarro2() e moverCarro2() são definidas para criar e controlar o movimento do carro inimigo.

Um temporizador (meuTimer) é usado para mover o carro inimigo periodicamente.

Final do Jogo:

Quando as vidas do jogador se esgotam, o jogo é encerrado e uma imagem de "game over" é exibida no centro da tela.

Este código cria um jogo simples baseado em toque onde o jogador controla o carro do jogador para evitar colidir com carros inimigos e coletar pontos.
